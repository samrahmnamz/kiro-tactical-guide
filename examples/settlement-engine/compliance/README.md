- **SOX Section 404** - Internal Controls Over Financial Reporting
- **FDIC Guidance** - Cloud Computing and Data Residency
- **Federal Reserve Policy** - Payment System Risk Management
- **OCC Bulletin 2013-29** - Third-Party Relationships

## Compliance Automation

The Settlement Engine uses Kiro's regulatory compliance toolkit to automate control enforcement:

### Automated Controls

- **`deployment-window.yaml`** - Enforces market hours restrictions
- **`require-approvals.yaml`** - Validates CAB approval presence
- **`validate-iam.yaml`** - Prevents wildcard IAM permissions
- **`region-config.yaml`** - Enforces us-east-1 data residency

### Manual Controls

- **CAB Approval Process** - Human review and approval of changes
- **Emergency Override Authorization** - VP+ approval for market hours deployments
- **Quarterly Rollback Drills** - Validation of rollback procedures
- **Annual Control Testing** - SOX 404 control effectiveness testing

## Compliance Evidence Collection

### Automated Evidence

The Settlement Engine automatically generates compliance evidence:

1. **Audit Logs** - All settlement actions with actor identity, timestamp, authorization
2. **CloudTrail Logs** - All AWS API calls with IAM principal, timestamp, source IP
3. **Step Functions History** - All workflow state transitions
4. **Hook Execution Logs** - All hook validations with pass/fail status

### Evidence Locations

```bash
# Audit logs (10-year retention)
s3://settlement-audit-logs/{year}/{month}/{day}/{settlementBatchId}/

# CloudTrail logs (exported from 90-day retention)
s3://cloudtrail-logs/AWSLogs/{accountId}/CloudTrail/{region}/{year}/{month}/{day}/

# Hook execution logs
~/.kiro/logs/hooks/

# Step Functions execution history
AWS Console → Step Functions → SettlementWorkflow → Execution history
```

## Quarterly Compliance Reports

The compliance team generates quarterly reports covering:

1. **Control Effectiveness** - Results of control testing
2. **Audit Trail Completeness** - Verification all actions logged
3. **Deployment Compliance** - Market hours deployment attempts (should be zero)
4. **Data Residency Verification** - Confirmation all data in us-east-1
5. **Exception Review** - Emergency overrides and justifications

Reports are stored in: `s3://compliance-reports/settlement-engine/{year}/Q{quarter}/`

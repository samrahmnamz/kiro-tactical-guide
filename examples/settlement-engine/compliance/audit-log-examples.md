# Audit Log Examples

## Overview

This document provides real-world examples of audit logs generated by the Settlement Engine for various operational scenarios. These audit logs satisfy SOX Section 404 requirements for traceability and demonstrate compliance with FDIC and Federal Reserve audit trail mandates.

**Audit Log Storage**: `s3://settlement-audit-logs/{year}/{month}/{day}/{settlementBatchId}/audit-{timestamp}.json`

**Retention Period**: 10 years (enforced by S3 lifecycle policy)

**Immutability**: S3 Object Lock with WORM (Write Once Read Many) compliance mode

## Audit Log Query Examples

### Query 1: Find all settlements initiated by a specific user

```sql
SELECT *
FROM settlement_audit_logs
WHERE auditEventType = 'settlement_initiated'
  AND actor.userId = 'alice@company.com'
  AND timestamp BETWEEN '2024-01-01' AND '2024-01-31'
ORDER BY timestamp DESC;
```

### Query 2: Find all emergency overrides in the last quarter

```sql
SELECT *
FROM settlement_audit_logs
WHERE authorization.emergencyOverride = true
  AND timestamp >= '2023-10-01'
ORDER BY timestamp DESC;
```

### Query 3: Verify segregation of duties compliance

```sql
SELECT 
  settlementBatchId,
  MAX(CASE WHEN auditEventType = 'settlement_initiated' THEN actor.userId END) AS initiator,
  MAX(CASE WHEN auditEventType = 'settlement_approved' THEN actor.userId END) AS approver,
  MAX(CASE WHEN auditEventType = 'settlement_executed' THEN actor.userId END) AS executor
FROM settlement_audit_logs
WHERE settlementBatchId = 'batch-abc123'
GROUP BY settlementBatchId;
```
